## Codebase Patterns
- Client uses baseUrl: "./src" in tsconfig.json for path aliases (import from "API", "hooks", "pages", etc.)
- @tailwindcss/vite ESM-only plugin fails with Vite 6 CJS mode — use @tailwindcss/postcss instead
- shadcn/ui path aliases use @/ prefix (e.g. @/components, @/lib) mapped in both tsconfig paths and vite resolve.alias
- Use `pnpm dlx shadcn@latest add <component> --yes` to add shadcn components
- Vite config mirrors these aliases via resolve.alias in vite.config.ts
- Environment variables use VITE_* prefix, accessed via import.meta.env.VITE_*
- Vite 6.x used (not 8.x) due to Node 20.10 on this machine; Vite 8 requires Node 20.19+
- Dev server runs on port 3000 with proxy to backend at localhost:5001
- Vite resolve aliases must be added for each src/ subdirectory (e.g. `stores: path.resolve(__dirname, "src/stores")`) — tsconfig baseUrl alone doesn't work for Vite
- Use `useAuthStore.getState()` for non-React contexts (API interceptors, refreshToken); use `useAuthStore(state => state.xxx)` inside React components/hooks
- Zustand stores use `skipHydration: true` for encrypted persistence — call `setEncryptionDigest(digest)` then `store.persist.rehydrate()` after login
- Zustand stores are in src/stores/ (auth.ts, chats.ts, ui.ts); encrypted middleware in src/stores/middleware/encrypted-storage.ts
